Overview

Akta is a prototype for secure, agent-to-agent interactions using DIDs and VCs.

Prototype Disclaimer: Akta is under active development. The API and functionality may change as the project evolves.

The Challenge

As AI agents become more autonomous, establishing trust, managing delegation, and verifying claims between agents becomes critical.

Trust

How can agents trust the identities and capabilities of other agents?

Delegation

How can we safely delegate permissions between systems?

Verification

How can we cryptographically verify claims about capabilities?

Core Concepts

Verifiable Identity with DIDs

Decentralized Identifiers provide cryptographically verifiable digital identities for agents.

Verifiable Capabilities

Capabilities and permissions are expressed as Verifiable Credentials that can be cryptographically verified.

Credential Store

Secure storage mechanism for managing and retrieving credentials across agent interactions.

End-to-End Integrity

Cryptographic assurance that credentials haven't been tampered with throughout the delegation chain.

Secure Delegation

Akta enables secure delegation of capabilities between agents through a verifiable process:

1

Issuance

Root authority issues verifiable credentials to agents

2

Storage

Credentials are securely stored in the agent's credential store

3

Delegation

Agents delegate subset of capabilities to other agents

4

Verification

Receiving agents verify the delegation chain cryptographically

Use Cases

Automated Workflows

Securely delegate tasks across multi-agent systems with verifiable authority.

Resource Sharing

Grant temporary access to resources with cryptographic proof of authorization.

Agent Ecosystems

Build complex systems of specialized agents with clear trust relationships.

Foundation for Protocols

Build on Akta for Agent-to-Agent (A2A) and Multi-Party Compute (MPC) protocols.
